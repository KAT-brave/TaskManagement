# タスク管理アプリ 要件定義

## 1. プロジェクト概要

- アプリ名: TaskManagement
- 概要: Trello風のタスク管理アプリ
- 対象ユーザー: 個人利用（自分のみ）
- 目的: RaiseTech 講座の課題として提出

## 2. プロジェクト背景・課題

### 背景
個人のタスク管理において、やるべきことの優先順位や進捗状況が把握しにくいという課題がある。

### 課題
- タスクがメモやノートに散在しており、一元管理できていない
- タスクの進捗状況（未着手・進行中・完了）が一目でわからない
- タスクの優先順位を柔軟に変更できない

### 解決策
Trello風のボード形式でタスクを視覚的に管理することで、進捗状況を一目で把握し、ドラッグ&ドロップで優先順位を柔軟に変更できるアプリを作成する。

## 3. 技術スタック

| 役割 | 技術 |
|---|---|
| バックエンド | Ruby on Rails |
| フロントエンド | HTML / CSS（Rails ビュー） |
| ドラッグ&ドロップ | SortableJS |
| データベース | MySQL または PostgreSQL |
| 認証 | Devise |

## 3. 機能要件

### 3-1. 認証機能

- ユーザー登録（名前・メールアドレス・パスワード）
- ログイン / ログアウト
- 自分のデータのみ表示・操作できる

### 3-2. ボード機能

- ボードの作成・編集・削除
- ボード一覧の表示

### 3-3. リスト機能

- リストの作成・編集・削除
- ボード内に複数のリストを表示

### 3-4. カード機能

- カードの作成・編集・削除
- タイトルと説明文・期限日を持つ

### 3-5. カード移動機能

- ドラッグ&ドロップでリスト間を移動できる

## 4. 非機能要件

- ログインしていないユーザーはアクセス不可
- レスポンシブ対応は対象外（PC のみ）

### 対応ブラウザ
- Google Chrome（最新版）
- Microsoft Edge（最新版）
- Safari（最新版）

### 性能要件
- ページの読み込みは3秒以内を目安とする
- タスク数の上限は1リストにつき100件まで

### アクセシビリティ
- キーボード操作には対応しない
- スクリーンリーダー対応は対象外

### レスポンシブ対応
- PC専用（スマートフォン・タブレットは対象外）

## 画面遷移図

### 未ログイン状態

```
トップ画面（/）
  ├── [新規登録] → ユーザー登録画面 → 登録成功 → ボード一覧画面
  └── [ログイン] → ログイン画面
                     ├── ログイン成功 → ボード一覧画面
                     └── [ゲストログイン] → ボード一覧画面
```

### ログイン済み状態

```
ボード一覧画面（/boards）
  ├── [+ 新規作成] → ボード作成画面 → 作成成功 → ボード詳細画面
  ├── [ボードをクリック] → ボード詳細画面
  │     ├── [編集] → ボード編集画面 → 更新成功 → ボード詳細画面
  │     ├── [削除] → 削除成功 → ボード一覧画面
  │     ├── [+ リスト追加] → リスト作成 → ボード詳細画面
  │     └── [カードをクリック] → カード詳細画面
  │           ├── [保存] → ボード詳細画面
  │           └── [削除] → ボード詳細画面
  └── [ログアウト] → トップ画面
```

## 各UI要素の振る舞い・エラー時の表示

### 各UI要素の振る舞い

- バリデーションエラー時は該当の入力欄の下に赤文字でメッセージを表示する
- 作成・更新・削除の成功時は画面上部にフラッシュメッセージを表示する
- 削除ボタンは確認ダイアログ（「本当に削除しますか？」）を表示してから実行する
- ドラッグ&ドロップ中はカードを半透明にして移動中であることを視覚的に示す

### エラー時の表示

| エラーの種類 | 表示内容 |
|---|---|
| 必須項目が未入力 | 「〇〇を入力してください」と入力欄の下に表示 |
| 文字数オーバー | 「〇〇文字以内で入力してください」と表示 |
| メール形式が不正 | 「メールアドレスの形式が正しくありません」と表示 |
| ログインに失敗 | 「メールアドレスまたはパスワードが正しくありません」と表示 |
| 存在しないページ | 404エラーページを表示する |

## ユースケース / 操作フロー

### タスクを作成して完了にするまでの流れ

1. ログイン画面からログインする
2. ボード一覧画面で対象のボードをクリックする
3. 「Todo」リストの「+ カード」をクリックする
4. タイトルと説明を入力して保存する
5. 作成されたカードをドラッグして「進行中」リストに移動する
6. タスクが完了したらカードを「完了」リストにドラッグして移動する

### ボードを新しく作る流れ

1. ボード一覧画面で「+ 新規作成」をクリックする
2. ボード名を入力して保存する
3. 作成されたボードの詳細画面に遷移する
4. 「+ リスト追加」でリストを作成する

### ゲストとしてログインする流れ

1. ログイン画面の「ゲストログイン」ボタンをクリックする
2. デモアカウントで自動ログインされる
3. ボード一覧画面に遷移する

## 6. デモアカウント

評価者がすぐに動作確認できるよう、デモアカウントを用意する。

| 項目 | 内容 |
|---|---|
| メールアドレス | demo@example.com |
| パスワード | password |

- ログイン画面に「ゲストログイン」ボタンを設置する
- ボタン1クリックでデモアカウントに自動ログインできる

## 7. データベース設計

### テーブルの関係

```
User（1） → Board（多）
Board（1） → List（多）
List（1） → Card（多）
```

### users テーブル（ユーザー）

| カラム名 | 型 | 説明 |
|---|---|---|
| id | integer | 主キー |
| name | string | ユーザー名 |
| email | string | メールアドレス |
| password_digest | string | パスワード（暗号化） |
| created_at | datetime | 作成日時 |
| updated_at | datetime | 更新日時 |

### boards テーブル（ボード）

| カラム名 | 型 | 説明 |
|---|---|---|
| id | integer | 主キー |
| user_id | integer | 作成したユーザーのID |
| name | string | ボード名 |
| created_at | datetime | 作成日時 |
| updated_at | datetime | 更新日時 |

### lists テーブル（リスト）

| カラム名 | 型 | 説明 |
|---|---|---|
| id | integer | 主キー |
| board_id | integer | 所属するボードのID |
| name | string | リスト名 |
| position | integer | 表示順 |
| created_at | datetime | 作成日時 |
| updated_at | datetime | 更新日時 |

### cards テーブル（カード）

| カラム名 | 型 | 説明 |
|---|---|---|
| id | integer | 主キー |
| list_id | integer | 所属するリストのID |
| title | string | カードタイトル |
| description | text | カードの説明 |
| due_date | date | 期限日 |
| position | integer | リスト内の表示順 |
| created_at | datetime | 作成日時 |
| updated_at | datetime | 更新日時 |

## 9. 制約・バリデーション

### users（ユーザー）

| 項目 | ルール |
|---|---|
| 名前 | 必須・50文字以内 |
| メールアドレス | 必須・重複不可・メール形式 |
| パスワード | 必須・6文字以上 |

### boards（ボード）

| 項目 | ルール |
|---|---|
| ボード名 | 必須・50文字以内 |
| ボード数 | 1ユーザーにつき最大20ボードまで |

### lists（リスト）

| 項目 | ルール |
|---|---|
| リスト名 | 必須・50文字以内 |

### cards（カード）

| 項目 | ルール |
|---|---|
| タイトル | 必須・100文字以内 |
| 説明 | 任意・1000文字以内 |
| カード数 | 1リストにつき最大100カードまで |
| 期限日 | 任意・過去の日付は設定不可 |

### UI・操作

| 項目 | 内容 |
|---|---|
| エラーメッセージ | バリデーション失敗時に該当項目の近くに表示する |
| フラッシュメッセージ | 作成・更新・削除の成功・失敗時に画面上部に表示する |
| ログイン必須 | 未ログインのユーザーがアクセスした場合はログイン画面にリダイレクトする |
