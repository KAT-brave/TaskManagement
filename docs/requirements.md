# タスク管理アプリ 要件定義

## 1. プロジェクト概要

- アプリ名: TaskManagement
- 概要: Trello風のタスク管理アプリ
- 対象ユーザー: 個人利用（自分のみ）
- 目的: RaiseTech 講座の課題として提出

---

## 2. プロジェクト背景・課題

### 背景
個人のタスク管理において、やるべきことの優先順位や進捗状況が把握しにくいという課題がある。

### 課題
- タスクがメモやノートに散在しており、一元管理できていない
- タスクの進捗状況（未着手・進行中・完了）が一目でわからない
- タスクの優先順位を柔軟に変更できない

### 解決策
Trello風のボード形式でタスクを視覚的に管理することで、進捗状況を一目で把握し、ドラッグ&ドロップで優先順位を柔軟に変更できるアプリを作成する。

---

## 3. 非機能要件

- ログインしていないユーザーはアクセス不可
- レスポンシブ対応は対象外（PC のみ）

### 対応ブラウザ
- Google Chrome（最新版）
- Microsoft Edge（最新版）
- Safari（最新版）

### 性能要件
- ページの読み込みは3秒以内を目安とする
- タスク数の上限は1リストにつき100件まで

### アクセシビリティ
- キーボード操作には対応しない
- スクリーンリーダー対応は対象外

### レスポンシブ対応
- PC専用（スマートフォン・タブレットは対象外）

---

## 4. デモアカウント

評価者がすぐに動作確認できるよう、デモアカウントを用意する。

| 項目 | 内容 |
|---|---|
| メールアドレス | demo@example.com |
| パスワード | password |

- ログイン画面に「ゲストログイン」ボタンを設置する
- ボタン1クリックでデモアカウントに自動ログインできる

---

## 5. 制約・バリデーション

### users（ユーザー）

| 項目 | ルール |
|---|---|
| 名前 | 必須・50文字以内 |
| メールアドレス | 必須・重複不可・メール形式 |
| パスワード | 必須・6文字以上 |

### boards（ボード）

| 項目 | ルール |
|---|---|
| ボード名 | 必須・50文字以内 |
| ボード数 | 1ユーザーにつき最大20ボードまで |

### lists（リスト）

| 項目 | ルール |
|---|---|
| リスト名 | 必須・50文字以内 |

### cards（カード）

| 項目 | ルール |
|---|---|
| タイトル | 必須・100文字以内 |
| 説明 | 任意・1000文字以内 |
| カード数 | 1リストにつき最大100カードまで |
| 期限日 | 任意・過去の日付は設定不可 |

### UI・操作

| 項目 | 内容 |
|---|---|
| エラーメッセージ | バリデーション失敗時に該当項目の近くに表示する |
| フラッシュメッセージ | 作成・更新・削除の成功・失敗時に画面上部に表示する |
| ログイン必須 | 未ログインのユーザーがアクセスした場合はログイン画面にリダイレクトする |
