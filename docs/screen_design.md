# 画面設計

## 1. 画面一覧

| 画面名 | URL | 説明 |
|---|---|---|
| トップ画面 | / | ログイン・登録へのリンク |
| ユーザー登録画面 | /users/sign_up | 名前・メール・パスワードを入力 |
| ログイン画面 | /users/sign_in | メール・パスワードを入力 |
| ボード一覧画面 | /boards | 自分のボード一覧を表示 |
| ボード詳細画面 | /boards/:id | ボード内のリスト・カードを表示 |
| ボード作成画面 | /boards/new | ボード名を入力して作成 |
| ボード編集画面 | /boards/:id/edit | ボード名を編集 |
| カード詳細画面 | /boards/:board_id/cards/:id | カードのタイトル・説明を表示・編集 |

---

## 2. 画面遷移図

### 未ログイン状態

```
トップ画面（/）
  ├── [新規登録] → ユーザー登録画面 → 登録成功 → ボード一覧画面
  └── [ログイン] → ログイン画面
                     ├── ログイン成功 → ボード一覧画面
                     └── [ゲストログイン] → ボード一覧画面
```

### ログイン済み状態

```
ボード一覧画面（/boards）
  ├── [+ 新規作成] → ボード作成画面 → 作成成功 → ボード詳細画面
  ├── [ボードをクリック] → ボード詳細画面
  │     ├── [編集] → ボード編集画面 → 更新成功 → ボード詳細画面
  │     ├── [削除] → 削除成功 → ボード一覧画面
  │     ├── [+ リスト追加] → リスト作成 → ボード詳細画面
  │     └── [カードをクリック] → カード詳細画面
  │           ├── [保存] → ボード詳細画面
  │           └── [削除] → ボード詳細画面
  └── [ログアウト] → トップ画面
```

### カード編集モーダル

カードをクリックするとモーダルが開く。別画面への遷移は行わない。

```
+------------------------------------------+
|  カードを編集                       [✕]  |
|                                          |
|  タイトル                                |
|  [____________________________]          |
|                                          |
|  説明                                    |
|  [____________________________]          |
|  [____________________________]          |
|                                          |
|  期限日                                  |
|  [____________________________]          |
|                                          |
|  優先度                                  |
|  [未設定 ▼]  ← 高 / 中 / 低             |
|                                          |
|  [保存]  [キャンセル]                    |
|  ─────────────────────────              |
|  🗑 このカードを削除                     |
+------------------------------------------+
```

- 優先度バッジはカード上に 🔴高 / 🟡中 / 🟢低 で表示する
- 削除ボタンは確認メッセージに切り替わってから実行する（2段階確認）

## 4. UIデザイン方針

- Apple風のミニマル・ライトデザイン
- 背景色: `#f5f5f7`（薄いグレー）、カード・パネル: 白（`#ffffff`）
- アクセントカラー: `#0066cc`（ブルー）
- テキスト: `#1d1d1f`（ダーク）、サブテキスト: `#6e6e73`（グレー）
- フォント: Inter（サンセリフ）
- ボタン形状: ピル型（角丸 `border-radius: 980px`）
- カード・パネルの角丸: `14px`〜`18px`
- ヘッダー: すりガラス効果（`backdrop-filter: blur`）

### ボードカードの色

| ボード例 | 背景色 | 文字色 |
|---|---|---|
| 仕事 | 白（`#ffffff`） | 黒（`#1d1d1f`） |
| プライベート | グレー（`#a0a0a8`） | 白（`#ffffff`） |
| 勉強 | 濃いグレー（`#3a3a3e`） | 明るいグレー（`#f5f5f7`） |

## 5. UI要素の振る舞い・エラー時の表示

### UI要素の振る舞い

- バリデーションエラー時は該当の入力欄の下に赤文字でメッセージを表示する
- 作成・更新・削除の成功時は画面上部にフラッシュメッセージを表示する
- 削除ボタンは確認ダイアログ（「本当に削除しますか？」）を表示してから実行する
- ドラッグ&ドロップ中はカードを半透明にして移動中であることを視覚的に示す

### エラー時の表示

| エラーの種類 | 表示内容 |
|---|---|
| 必須項目が未入力 | 「〇〇を入力してください」と入力欄の下に表示 |
| 文字数オーバー | 「〇〇文字以内で入力してください」と表示 |
| メール形式が不正 | 「メールアドレスの形式が正しくありません」と表示 |
| ログインに失敗 | 「メールアドレスまたはパスワードが正しくありません」と表示 |
| 存在しないページ | 404エラーページを表示する |
